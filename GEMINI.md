# Инструкции для Gemini CLI

## Ограничения на запуск кода:
1. Запрещено запускать полные циклы обучения.
2. При тестировании кода обучения ВСЕГДА ограничивать выборку (например, `num_samples=100`) и количество итераций (например, `max_epochs=1` или `steps=5`).
3. Цель любого запуска — только проверка синтаксиса и корректности работы с данными (smoke test).

## Окружение и GPU:
1. На сервере есть GPU: RTX 4090 (индекс 0) и RTX 4080 (индекс 1).
2. Для запуска кода всегда используй CUDA_VISIBLE_DEVICES=0 /home/levos/miniconda3/envs/baikal/bin/python <НАЗВАНИЕ СКРИПТА>. Если скрипт упадет с ошибкой, говорящей о том, что видеокарта 0 занята, попробуй использоать CUDA_VISIBLE_DEVICES=1, а если и это не удастся - сообщи о том, что обе видеокарты заняты. 
3. Для быстрых тестов логики (не нейросетевых) лучше использовать CPU, чтобы не занимать видеокарты.

## Структура проекта и организация экспериментов:
- Все новые исследования проводятся в директории `~/experiments/`.
- Для каждого эксперимента создается отдельная подпапка с **понятным смысловым названием** (например, `2026-01-27_gcn_hdf5_test`).
- **Обязательная структура внутри папки эксперимента:**
    - `src/` — все исходные файлы с кодом (.py, .ipynb).
    - `plots/` — сохраненные графики, гистограммы и визуализации.
    - `README.md` — краткое описание: цель эксперимента, используемые параметры и промежуточные выводы.
